利用hexo搭建个人博客

花一点点时间就可以搭建一个属于自己的博客了，真的只是一点点 如果一切顺利的话。下面给大家演示一下（本文针对OSX系统，即Mac用户）

1. 注册Github帐号及创建仓库

作为一个程序员，没有Github帐号不应该吧。作为全球最大的交友网站，你值得拥有！没有的话也没关系，花几分钟时间注册一下：https://github.com.
注册过程和一般帐号的注册过程没什么区别，按照引导即可。注册页面大概长这样

然后创建一个仓库（repository）来存储我们的网站，点击首页任意位置出现的 New repository 按钮创建仓库。

2. 安装Git Nodejs Hexo

下面安装我们需要借助的几个工具：

1.安装git

对于大多数开发者 应该都已经安装了。作为iOS开发者我推荐利用Xcode：打开Xcode–>Preferences（快捷键 Command + ，（逗号））点击Downloads选项，下载Command Line Tools。安装完以后可以在终端输入 git -v来查看当前版本

2.安装 Nodejs

安装Nodejs之前需要先安装nvm(Nodejs的版本管理器)
$ curl https://raw.github.com/creationix/nvm/master/install.sh | sh
安装完成之后重启终端 再终端输入
$ nvm install 4
这样Nodejs就安装完了 同样可以使用 nodejs -v 查看版本信息

3.安装Hexo

$ sudo npm install hexo-cli -g
(略作说明：$开头的都表示在终端中输入)
现在工具都安装完了 万事俱备只欠东风。

3.博客创建及设置操作

1. 创建博客

$ hexo init username.github.io
意思是利用hexo创建一个文件夹，以后有关于你的博客的所有东西都在这里面，username.github.io是文件夹的名字，可以命名为你的博客域名或直接叫Blog。创建完大概是这样的一个文件夹

2.设置theme

炫酷的界面效果就靠它，或者说装逼主要靠它。hexo提供了非常丰富的主题选择:https://hexo.io/themes/
选好之后将操作路径切换至刚刚创建的那个文件夹 然后安装主题
$ cd username.github.io $ git clone https://github.com/iissnan/hexo-theme-next themes/next
上面用next做了一个示范，next是使用者最多的一个主题，我自己用的是typing主题，只需要将上面的next改为typing即可，起他主题一样的道理。安装完之后就可以看到在在themes文件夹下面会有一个刚刚创建的主题名的文件夹，对于该主题的相关操作都在这里面

3.更改配置

打开文件位置username.github.io/_config.yml修改几个键值对，下面把几个必须设置的列出来按需求修改，记得保存， 还有注意配置的键值之间一定要有空格
title: EndaKun's Den //你博客的名字 author: HuangChenkun //你的名字 language: zh-Hans //语言 中文 theme: next //刚刚安装的主题名称 deploy: type: git //使用Git 发布 repo: https://github.com/username/username.github.io.git // 刚创建的Github仓库
其他键值可以不修改 看个人需求

写博客并发布

现在就真的是万事俱备只欠东风了，下面我们就可以来写博客了，主要是介绍几个hexo指令

1.创建文章

$ hexo new "My New Post"
通过上一句即可创建一个标题为My New Post的文章了。略作说明：创建的post以及之前所有的post都在source文件夹的_posts里。创建完之后会出现一个 My New Post.md的文件，需要用 Markdown （一种轻量级标记语言，专门用于写作的）来撰写

2.本地测试

$ hexo s
本地测试的意思是在本地查看博客，安全方便，随时查看随时测试方便修改，。当然本地测试在后面第三步第四步后操作也可以 或者随时操作都可以

3.安装hexo-deployer-git自动部署发布工具

$ npm install hexo-deployer-git --save
这一步可能很多人会遗漏 不安装的话是没办法发布的

4.发布

发布的意思就是发布成功以后所有人都可以看到你的博客网站里
$ hexo clean && hexo g && hexo d
上面是三个指令 当然也可以三个指令分开来写。终端里提示发布成功之后就可以查看效果了。关于终端给新人一点小提示：左上角最左边显示一个点的话表示终端还在运行，要等一个点变成一个叉号就表示运行完毕了。然后我们就可以在网站输入 https://huangchenkun.github.io查看效果了，当然别忘了huangchenkun换成你自己在Github的用户名，huangchenkun是本宝宝的。
所有的操作介绍完毕 顺利的话十几分钟就可以搞定，大家可以试试
